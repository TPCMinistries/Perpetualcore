# Phase 24-01 Plan — RFP Health Endpoint Hardening

## Requirement

- ADMIN-04: `/api/health/rfp` returns JSON status with scraper last success, drift open, cron last run, and error rate.

## Scope

Harden the existing public `GET /api/health/rfp` endpoint. Keep it aggregate-only because uptime/status monitors poll it without authentication.

## Tasks

1. Add a testable cron-health summarizer for recent RFP cron executions.
2. Extend `/api/health/rfp` with:
   - latest successful scraper baseline
   - 24-hour RFP cron run counts
   - 24-hour cron failures, warnings, successes
   - 24-hour cron error rate
3. Preserve existing fields for downstream monitors and admin links.
4. Add focused unit tests for the error-rate calculation.
5. Smoke the route with local environment loaded.

## Out of Scope

- Wiring Better Stack/UptimeRobot externally. Code now provides the fields; external monitor setup remains an operator task unless credentials/API access are supplied.
- Admin source rerun controls. That belongs to ADMIN-03.
- FTUE checklist/empty states. Those remain separate Phase 24 slices.
