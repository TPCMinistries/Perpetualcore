# Phase 24-03 Plan — Source Health Manual Rerun

## Requirement

- ADMIN-03: Operator sees source health/drift with a manual "rerun now" control.

## Scope

Add a source-scoped rerun path to the existing `/admin/rfp` source readiness table. Reuse existing ingest orchestrators rather than duplicating source parser logic.

## Tasks

1. Add optional source filters to:
   - `runFederalIngest`
   - `runStateCityIngest`
2. Add source support detection:
   - federal API sources
   - state/city scraper sources
   - SBIR schema source `sbir` maps to fetcher source `sbir_gov`
3. Add `rerunRfpSource(source)` admin helper:
   - runs only that source
   - scores newly upserted opportunities
   - logs a `cron_executions` row as `rfp-manual-source-rerun:{source}`
4. Add a server action to `/admin/rfp`.
5. Add a Rerun column to source readiness.
6. Show `No runner` for planned, blocked, licensed, curated, and manual-import sources.

## Verification

- Unit tests for source support/mapping.
- Focused ESLint.
- Runtime import check with local env loaded.

## Out of Scope

- Running a live parser rerun from this implementation session. That would write production baselines/drift rows and should be done deliberately by the operator for a chosen source.
- Parser fixes for the currently open drift events.
