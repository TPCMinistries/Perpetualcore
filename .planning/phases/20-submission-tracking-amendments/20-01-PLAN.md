# Phase 20-01 Plan — Canonical No-Bid Lifecycle Status

**Phase:** 20 — Submission Tracking & Amendments
**Requirement:** SUBMIT-02
**Date:** 2026-06-11

## Goal

Align the proposal lifecycle vocabulary with the roadmap requirement: users should record and see `draft`, `submitted`, `won`, `lost`, and `no-bid` outcomes. Keep legacy `withdrawn` rows readable for backward compatibility.

## Scope

- Add canonical `no_bid` to the proposal status API allowlist.
- Update proposal and pursuit UI labels so users see `No-bid`, not raw DB tokens.
- Keep `withdrawn` accepted and rendered as `No-bid` for existing records.
- Update the live `rfp_proposals.status` CHECK constraint.
- Add/adjust RLS coverage for `PATCH status='no_bid'`.

## Out of Scope

- Amendment polling and diffing (`SUBMIT-03`, `SUBMIT-04`).
- Win/loss debrief capture (`LEARN-01`).
- Relaxing the submit-readiness gate for `submitted`.

## Verification

- Unit tests for existing submission exports/readiness.
- Focused RLS test for `PATCH status='no_bid'`.
- Live DB constraint inspection.
