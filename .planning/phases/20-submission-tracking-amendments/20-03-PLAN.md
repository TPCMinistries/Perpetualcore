# Phase 20-03 Plan — Amendment Snapshots and Diffing

**Phase:** 20 — Submission Tracking & Amendments
**Requirements:** SUBMIT-03, SUBMIT-04
**Date:** 2026-06-11

## Goal

Persist solicitation snapshots for active pursuits, detect amendments/addenda by comparing the latest capture against the prior snapshot, and surface the diff to users.

## Scope

- Add org-scoped snapshot and amendment tables with RLS.
- Build deterministic snapshot hashing and diff logic.
- Add a CRON_SECRET-protected amendment monitor route.
- Add an authenticated amendments read endpoint.
- Show recent amendments in the Discovery detail pane.
- For material amendments, create a pursuit decision-log entry and a critical submission task to re-run compliance/fit checks before submission.

## Out of Scope

- Source-specific scraper rewrites.
- Email/Telegram/Discord channel fan-out for amendment alerts.
- Automatic LLM reviewer reruns.

## Verification

- Unit tests for material amendment diffing.
- ESLint over touched amendment files/routes/UI.
- Live DB table/RLS verification after migration.
