# Phase 26 — Best-in-Market Readiness Plan

Status anchor: 2026-06-16. Target is not "deployable"; target is best-in-market RFP/capture operations across functionality, look, and user experience.

## Definition of 100% Ready

RFP Engine is 100% ready when a new buyer can land on the public site, start a trial, create an org, see real live opportunities, understand why each opportunity matches, make a bid/no-bid decision, draft grounded proposal sections, run compliance/reviewer checks, export a submission packet, manage billing, and trust the system operationally.

The product must beat the practical workflow of Instrumentl, GovWin, OpenGrants, Submittable, generic ChatGPT/Claude, and capture consultants for the beachhead buyer: mission-driven orgs and small capture teams that cannot hire a full capture department.

## Track A — Launch-Critical Reliability

Goal: no hidden build or runtime failure paths.

1. Fix all lint errors and reduce warning noise that masks real regressions.
2. Run and pass `npm run lint`, `npm run type-check:rfp`, `npm run test:run`, and production build; split or repair full monorepo `npm run type-check` before broad launch.
3. Replace build-time ignores with enforced CI gates, or document why Vercel skips are acceptable only because CI blocks merge.
4. Fix dependency vulnerabilities: no critical or high production-risk issues without documented exception; reduce or document moderate transitive issues.
5. Fix branch tracking and merge posture so deploys come from stable `main`.

## Track C — Best-in-Market Product Workflow

Goal: complete capture operations, not a marketing shell.

1. Capture Console: single authenticated home for opportunity inbox, pursuit board, proposal workroom, alerts, and next actions.
2. Opportunity Inbox: filters for source, deadline, fit, award size, geography, eligibility, set-aside, funding type, and saved searches.
3. Opportunity Detail: fit reasoning, disqualifiers, eligibility parser, deadline timeline, attachments, rubric, and recommended next action.
4. Bid/No-Bid: stakeholder notes, decision owner, due date, risk, and audit trail.
5. Proposal Workroom: section editor, vault citations, `[VERIFY]` flags, reviewer comments, regenerate-by-section, and version history.
6. Compliance Matrix: page limits, attachments, budget math, eligibility, deadline/timezone, AI disclosure, and final readiness score.
7. Export Bundle: proposal, compliance matrix, citations, manifest, packet CSV, audit trail, and submission checklist.
8. Alerts: saved search alerts, deadline reminders, material amendment alerts, and weekly digest.
9. Billing: trial, upgrade, portal, entitlement enforcement, and plan copy alignment.

## Track D — Look, UX, and Accessibility

Goal: Stripe/Linear-grade operating software for capture work.

1. Redesign authenticated RFP surfaces around dense, calm, scan-friendly operations.
2. Use product screenshots/live product UI on public pages, not abstract claims.
3. Add polished empty states, skeletons, and loading states for every critical workflow.
4. Add command-menu navigation and keyboard-friendly workflows.
5. Run responsive QA for desktop, tablet, and mobile review paths.
6. Run accessibility checks for keyboard focus, labels, color contrast, headings, and form errors.
7. Remove stale/internal-facing copy from public pages.

## Track E — SEO, Trust, and Market Position

Goal: credible acquisition surface and buyer confidence.

1. Serve `robots.txt` and `sitemap.xml` correctly on `rfp.perpetualcore.com`.
2. Ensure canonical URLs, Open Graph, Twitter, structured data, and sitemap entries align to the RFP subdomain.
3. Strengthen Trust page: RLS, audit logs, AI limitations, human-review requirement, data retention, and source compliance.
4. Build comparison pages against Instrumentl, GovWin, OpenGrants, Submittable, ChatGPT/Claude, and consultants.
5. Build an ROI calculator using missed opportunities, staff hours, consultant savings, and win probability.
6. Create a polished demo org with realistic sample data and sales walkthrough.
7. Publish at least one dogfood case study or internal proof note from Uplift/IHA/TPC.
