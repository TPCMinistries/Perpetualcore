**1a. `lib/rfp/import/fetch-url.ts`**:

```ts
export interface FetchedContent {
  url: string;
  contentType: 'html' | 'pdf' | 'docx' | 'unknown';
  text: string;          // Plain text after parse
  bytes?: Buffer;        // Original binary for PDFs/DOCX (optional)
  size: number;
}

export async function fetchUrlContent(url: string): Promise<FetchedContent>
```

Behavior:
  - Validate URL parses, scheme is http/https. Throw `URL_INVALID` otherwise.
  - Fetch with timeout 15s, max body 10 MB (use `AbortController` and stream the body, abort if size exceeds). User-Agent same as scrapers.
  - Branch on `Content-Type`:
    - `text/html` → use cheerio if installed, else simple regex strip of `<script>/<style>` then strip tags. Project already has `pdf-parse` and `docx` per Phase 4 SUMMARYs — check `package.json` to see if cheerio is there too. If not, do NOT add it; use a regex-based fallback.
    - `application/pdf` → run through `pdf-parse` to get text.
    - `application/vnd.openxmlformats-officedocument.wordprocessingml.document` → run through `docx` package (the same `docx`/`mammoth` already installed for Phase 6).
    - Other → return contentType `unknown` with raw text best-effort.
  - Return text capped at 100 KB (truncate if more). Logs a warning on truncation.

**1c. `lib/rfp/import/job-store.ts`** — Upstash Redis-backed persistence (REQUIRED — no module-Map fallback):

```ts
import { Redis } from '@upstash/redis';

const JOB_TTL_SECONDS = 60 * 60; // 1 hour

// Construct ONCE at module load, exactly like lib/rate-limit/index.ts:48-57.
// Both UPSTASH_REDIS_REST_URL and UPSTASH_REDIS_REST_TOKEN MUST be set in
// any environment that runs Quick Import (dev + preview + prod). Failure to
// configure Redis is a startup error — there is NO Map fallback because
// module-scope Maps do not survive across Vercel lambda invocations
// (POST sets state on lambda A, GET polls lambda B → 404 → user has no
// way to know whether import succeeded → breaks Success Criterion 5).
function getRedis(): Redis {
  const url = process.env.UPSTASH_REDIS_REST_URL?.trim();
  const token = process.env.UPSTASH_REDIS_REST_TOKEN?.trim();
  if (!url || !token) {
    throw new Error(
      'Quick Import requires UPSTASH_REDIS_REST_URL and UPSTASH_REDIS_REST_TOKEN. ' +
      'See lib/rate-limit/index.ts for the same client construction pattern.'
    );
  }
  return new Redis({ url, token });
}

const redis = getRedis();

export { JOB_TTL_SECONDS };

export async function writeJob(jobId: string, job: ImportJob): Promise<void> {
  await redis.set(`rfp:import:${jobId}`, JSON.stringify(job), { ex: JOB_TTL_SECONDS });
}

export async function readJob(jobId: string): Promise<ImportJob | null> {
  const raw = await redis.get(`rfp:import:${jobId}`);
  if (!raw) return null;
  // Upstash REST returns parsed JSON when value was set as JSON string; handle both.
  return typeof raw === 'string' ? JSON.parse(raw) as ImportJob : raw as ImportJob;
}
```

The ImportJob interface lives in `lib/rfp/import/run.ts` and is imported here (or move to a shared `lib/rfp/import/types.ts` if circular imports arise).

**1d. `lib/rfp/import/run.ts`**:

```ts
export type ImportStep = 'fetching' | 'parsing' | 'scoring' | 'done' | 'error';

export interface ImportJob {
  jobId: string;
  userId: string;
  orgId: string;
  step: ImportStep;
  status: 'in_progress' | 'success' | 'failure';
  error?: string;
  opp_id?: string;
  needs_review?: boolean;
  created_at: string;
  updated_at: string;
}

export async function runQuickImport(opts: { url: string; orgId: string; userId: string; jobId: string }): Promise<void>
```

Status persistence: ALWAYS via `lib/rfp/import/job-store.ts` (writeJob/readJob). The Redis key TTL is 1 hour. Document at top of file:

> "Quick Import job state is stored in Upstash Redis only. Module-scope Maps are explicitly NOT used — they do not survive across Vercel lambda invocations (POST → set on lambda A, GET → poll lambda B → 404). For horizontal scaling concerns beyond MVP volume (e.g., persistent jobs across deploys, history, retries), see deferred-items.md QUICK-IMPORT-QUEUE-DURABILITY."

Steps:
  1. Set `step='fetching', status='in_progress'`. Call fetchUrlContent. On error → step='error', status='failure', error message.
  2. `step='parsing'`. Call extractOpportunityFromText.
  3. Build OpportunityInput via normalizeOpportunity:
     - source = 'foundation_url' (matches schema enum)
     - source_id = SHA-256 hash of the URL (deterministic, lets us upsert on re-import)
     - url = the input URL
     - needs_review = (confidence !== 'high') OR (extracted fields lack title)
     - If title missing, default to 'Imported: ' + URL hostname for visibility.
  4. Upsert into rfp_opportunities via createAdminClient with onConflict 'source,source_id'. Capture `id`.
  5. `step='scoring'`. Call `scoreNewOpportunitiesForAllActiveOrgs([opp_id])` from lib/rfp/scoring/recompute.ts. (This scores for ALL active orgs, including the importing user's. That's correct — Quick Import opps should score for everyone like federal feeds.)
  6. `step='done', status='success', opp_id, needs_review`.
  7. Each step writes the updated ImportJob to Redis via writeJob. Always set `updated_at = new Date().toISOString()`.

Error policy: any thrown error inside fetch/extract/score → set step='error', status='failure', error=err.message. NEVER throw out of runQuickImport; the orchestrator handles errors as terminal states.
